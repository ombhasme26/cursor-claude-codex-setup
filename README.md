# Cursor, Claude Code and Codex Setup

## Overview

This repository documents my attempt to set up Cursor IDE, Claude Code, Codex, and GitHub version control.

## Tools Installed

- Cursor IDE
- GitHub account and public GitHub repository
- Git integration through Cursor
- Cursor Plugins/Marketplace access

## Steps Completed

1. Installed Cursor IDE on Windows.
2. Created and signed in to my Cursor account.
3. Created a public GitHub repository named `cursor-claude-codex-setup`.
4. Opened the repository in Cursor.
5. Opened the Customize section and accessed the Cursor Plugins/Marketplace page.
6. Searched for `Claude Code` in the marketplace.
7. Checked Cursor’s New Agent section for available AI providers.
8. Created this README.md file to document the setup process.
9. Added screenshot evidence of the marketplace search result.
10. Committed and pushed this repository to GitHub.

## Issues Encountered and Solutions

### Claude Code extension not found

I searched for `Claude Code` in Cursor’s current Plugins/Marketplace section. No matching result was available in the marketplace shown in my installed version of Cursor.

**Solution:** I verified that this Cursor version uses its own Plugins and New Agent interface. I did not install any unofficial third-party extension from an unknown publisher.

### Codex extension availability

I also checked Cursor’s marketplace and New Agent interface for Codex. The requested Codex extension was not available as a separate official plugin in the marketplace shown in my installed Cursor version.

**Solution:** I documented the limitation and checked the built-in New Agent section for available model providers.

### Model subscription or API access

I do not currently have a paid Claude subscription, OpenAI API credits, or a paid AI model subscription.

**Solution:** I completed all setup steps that do not require payment, including installing Cursor, checking the plugin marketplace, creating a public GitHub repository, opening the repository in Cursor, documenting the process, and pushing the project to GitHub.
